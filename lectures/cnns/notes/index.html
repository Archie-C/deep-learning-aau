Convolutional Neural Networks

Introduction

Computer vision is a subsection of machine learning that uses the structured data in images to perform visual reasoning. Convolutional neural networks first introduced in the 1980s, are now the most widely used architecture for computer vision tasks, taking over from 3D perspective geometry-based approaches.

In these notes we'll cover the mathematics behind convolutional neural networks, for more details about the actual implementation see this. We omit some of the formal notation for brevity and to help understanding, if you want to see a more formal definition see: Deep Learning: Foundations and Concepts

Computer Vision Tasks

To give you a glimpse of what you can do with what we will cover, here are some of the most common computer vision tasks:

  • Classification: For example, classifying images of freckles and moles into benign and malignant categories.

  • Object Detection:Identifying and locating objects within images, such as in autonomous driving.

  • Image Segmentation: Dividing an image into multiple groups (segments) for example, sky, grass, trees.

  • Caption Generation: Creating descriptive captions for images.

  • Synthesis: Generating new images from existing ones or from text descriptions.

  • Inpainting: Filling in missing or damaged parts of images. This is particularly useful in photoshop.

  • Style Transfer: Transforming the style of one image to match another.

  • Super Resolution: Enhancing the resolution of low-quality images. Great for restoring old photographs.

  • Depth Estimation: Predicting the distance of objects from the camera in an image. Again, crucial in autonomous driving.

  • Scene Reconstruction: Reconstructing a 3D scene from 2D images.

Image Data

Light is a continuous signal, but when we capture images digitally we have to sample this signal at discrete intervals, both in space and in intensity. The result is a grid of pixels, where each pixel has a value representing the intensity of light at that point. Typically, the intensity is represented as a value between 0 and 255 (an 8-bit unsigned integer), where 0 represents black and 255 represents white. That is for black and white images, for colour images we have three channels (red, green and blue), each with its own intensity value.

What this means is we can represent an image as a 3D array of shape:

\[ (height, width, channels) \]

If we consider a 1080p image, which has a resolution of 1920x1080 pixels, and 3 color channels (RGB), the shape of the array would be:

\[ (1080, 1920, 3) \]

This means that a single 1080p image contains 1080 * 1920 * 3 = 6,220,800 values (or pixels) that we can use as input to our neural network. This is a lot of data, and way too many inputs for our traditional neural networks.

Luckily, images are highly structured!

We can test this, if we take an image of a cat, and we randomly shuffle the pixels, we get a completely different image, that doesn't look at all natural. It just looks like random noise.

This means there must be some local structure in the image that we can exploit, and reduce the number of parameters we need to learn.
