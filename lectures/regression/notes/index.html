Regression

Introduction

Linear regression is perhaps the most fundamental machine learning algorithm. It underpins many of the concepts in this course and it's a great starting model to try as a benchmark on a new problem.

In these notes we'll cover the mathematics behind linear regression, for more details about the actual implementation see this. We omit some of the formal notation for brevity and to help understanding, if you want to see a more formal definition see: Deep Learning: Foundations and Concepts

Prerequisites

Before we get into the nitty gritty details, make sure you have a good understanding of the following:

  • Basic linear algebra

  • Calculus (derivatives and optimization)

  • Probability and statistics

The Model

Below we have some data from a fictional school experiment where we measured the height of some kids across year groups. A common task in school would be to get students to draw a line of best fit to model the relationship.

Now obviously we can see that there is a clear linear relationship between the age and the height of the kids, and we can draw a line of best fit. But the question is, how do we find the best line?

In school you probably just drew a line that looked good, but in machine learning we need a more rigorous way to find the optimal line. We can't just go around eyeballing lines of best fit for every problem we want to solve.

Defining the line

You should be familiar with the equation of a line from school, which is given by:

\[ y = mx + c \]

where \(m\) is the slope and \(c\) is the y-intercept.

But we need a more general definition because if we decided we wanted to model a relationship between multiple variables we don't want to have to keep incremementing the letter we use for the slope and intercept. So instead let's rewrite the equation as:

\[ y = w_1x + w_0 \]

Fitting the line

Now that we have the equation of the line defined, we need to find the optimal values for \(w_1\) and \(w_0\) that best fit our data.

To do this we define a loss function that measures how well our close our line is to the data points. We can then minimise this function to find the optimal parameters. For this example the loss function is quite trivial, we just want to minimise the distance between the line and the data points.

In practice, we will use Least Squares as our loss function. Which is given by:

\[ L(w_0, w_1) = \sum_{i=1}^{N} (y_i - (w_1x_i + w_0))^2 \]

(NOTE: we don't derive the probabilistic justification here)

Nicely, the minimisation of this function has a closed form solution, which means we can find the optimal parameters without having to use gradient descent of any other iterative optimsation tricks.

Since we have two parameters, we want to find the minimum of this function with respect to both \(w_0\) and \(w_1\). We can do this by taking the partial derivatives of the loss function with respect to each parameter and setting them to zero.

\[ \frac{\partial L}{\partial w_0} = -2 \sum_{i=1}^{N} (y_i - (w_1x_i + w_0)) = 0 \] \[ \frac{\partial L}{\partial w_1} = -2 \sum_{i=1}^{N} x_i (y_i - (w_1x_i + w_0)) = 0 \]

These are two simulaneous equations that we can solve by first dropping the -2 and then distributing the summation:

\[ \sum_{i=1}^{N} y_i - w_1 \sum_{i=1}^{N} x_i - N w_0 = 0 \] \[ \sum_{i=1}^{N} x_i y_i - w_1 \sum_{i=1}^{N} x_i^2 - w_0 \sum_{i=1}^{N} x_i = 0 \]

Lets work on the first equation:

If we rearrange the first equation for \(w_0\):

\[ w_0 = \frac{1}{N} \sum_{i=1}^{N} y_i - w_1 \frac{1}{N} \sum_{i=1}^{N} x_i \]

note that \(\frac{1}{N} \sum_{i=1}^{N} x_i = \bar{x}\) and \(\frac{1}{N} \sum_{i=1}^{N} y_i = \bar{y}\) as they are just the sample means.

So we can rewrite this as:

\[ w_0 = \bar{y} - w_1 \bar{x} \]

Which is a much nicer form than what we started with. Now let's tackle the second equation. We can simplify the \(\sum_{i=1}^{N} x_i\) as \(N\bar{x}\) and then subsitute our expression for \(w_0\) in to give:

\[ \sum_{i=1}^{N} x_i y_i - w_1 \sum_{i=1}^{N} x_i^2 - \left( \bar{y} - w_1 \bar{x} \right) N\bar{x} = 0 \]

And then with some rearrangement, which I will leave out as an exercise. We are left with our two analytical solutions:

\[ w_1 = \frac{\sum_{i=1}^{N} (x_i - \bar{x})(y_i - \bar{y})}{\sum_{i=1}^{N} (x_i - \bar{x})^2} \] \[ w_0 = \bar{y} - w_1 \bar{x} \]

Applying this to our example

If we go ahead and apply these two equations to our school experiment data, we find that the optimal parameters are roughly \(w_1 = 6\) and \(w_0 = 80\) (rounded to integer values). I'd recommend you try this yourself using the data points from the chart above (hover to see the values). This means that our line of best fit is given by:

\[ y = 6x + 80 \]

And our loss (e.g our error between the line and the data points) is around 20 (using the rounded values). Looking at the plot below, it looks like a pretty good fit, the line is close to all of the data points.

Multiple Input Dimensions

So far we have only looked at the case where each data point has a single feature (i.e. it is just one number). But what if we have multiple features? For example, if we were trying to predict the price of a house, we might have features like the size of the house, the number of bedrooms, the location, etc. In this case, we can extend our linear regression model to handle multiple input dimensions:

\[ y(\mathbf{x}, \mathbf{w}) = w_0 + w_1 x_1 + w_2 x_2 + \cdots + w_n x_n \]

It is conceptually the same, each feature has a corresponding weight that we need to learn/find and a bias term \(w_0\) that allows us to shift the line up and down. Even though we can't easily visualise this in more than 3 dimensions, we can still find the optimal parameters using the same least squares function as before. However, the 2D formula was long enough, so if we write them out the same way as before we would have n simultaneous equations and it would take forever. This is where linear algebra steps in.

We can rewrite out model in vector form. First we define the input vector:

\begin{equation} \mathbf{x} = \begin{bmatrix} 1 \\ x_1 \\ x_2 \\ \vdots \\ x_n \end{bmatrix}, \quad \mathbf{\theta} = \begin{bmatrix} w_0 \\ w_1 \\ w_2 \\ \vdots \\ w_n \end{bmatrix} \end{equation}

Note the we have 1 as the first element of the input vector to account for the bias term. This means both vectors are length \(n+1\). We also transition to the more typical notation of \(\mathbf{\theta}\) for the weight vector. Our model can then be written as:

\[ y(\mathbf{x}, \mathbf{\theta}) = \mathbf{\theta}^\top \mathbf{x} \]

Then our least squares problem becomes:

\[ \mathbf{L}(\mathbf{\theta}) = \sum_{i=1}^{N} \left( y_i - \mathbf{\theta}^\top \mathbf{x}_i \right)^2 \]

Now when we have multiple data points we can stack them together into a design matrix and a target vector:

\[ \mathbf{X} = \begin{bmatrix} 1 & x_{11} & x_{12} & \cdots & x_{1m} \\ 1 & x_{21} & x_{22} & \cdots & x_{2m} \\ \vdots & \vdots & \vdots & \ddots & \vdots \\ 1 & x_{N1} & x_{N2} & \cdots & x_{Nm} \end{bmatrix}, \quad \mathbf{y} = \begin{bmatrix} y_1 \\ y_2 \\ \vdots \\ y_N \end{bmatrix} \]

Where \(N\) is the number of data points and \(m\) is the number of features. Using the this notation and the same weight vector as before, we can write the problem even more concisely:

\[ \mathbf{L}(\mathbf{\theta}) = \|\mathbf{y} - \mathbf{X} \mathbf{\theta}\|^2 \]

Before we find the minimum of this function we need to do some rearrangement. The first thing to note is that the above is just equal to the transpose of the difference multiplied by the difference:

\[ \mathbf{L}(\mathbf{\theta})= \left( \mathbf{y} - \mathbf{X} \mathbf{\theta} \right)^\top \left( \mathbf{y} - \mathbf{X} \mathbf{\theta} \right)\]

If we expand this out we get:

\[ \mathbf{L}(\mathbf{\theta}) = \mathbf{y}^\top \mathbf{y} - 2 \mathbf{\theta}^\top \mathbf{X}^\top \mathbf{y} + \mathbf{\theta}^\top \mathbf{X}^\top \mathbf{X} \mathbf{\theta} \]

Which we can then differentiate with respect to \(\mathbf{\theta}\) to find the minimum. If we use standard matrix calculus and take each term in turn, for the first term we have:

\[ \frac{\partial}{\partial \mathbf{\theta}} \mathbf{y}^\top \mathbf{y} = 0 \]

For the second term:

\[ \frac{\partial}{\partial \mathbf{\theta}} (- 2 \mathbf{\theta}^\top \mathbf{X}^\top \mathbf{y}) = - 2 \mathbf{X}^\top \mathbf{y} \]

And for the third term:

\[ \frac{\partial}{\partial \mathbf{\theta}} (\mathbf{\theta}^\top \mathbf{X}^\top \mathbf{X} \mathbf{\theta}) = 2 \mathbf{X}^\top \mathbf{X} \mathbf{\theta} \]

Putting these all together we end up with:

\[ \nabla_\theta \mathbf{L} = 2 \mathbf{X}^\top \mathbf{X} \mathbf{\theta} - 2 \mathbf{X}^\top \mathbf{y} = 0 \]

Which is actually rather trivial to rearrange for \(\mathbf{\theta}\):

\[ \mathbf{\theta} = (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{y} \]

And there we have it, the optimal parameters for linear regression with \(n\) input dimensions. Given the size of the problem, I don't advocate for doing this by hand, but it is an interesting exercise to code up.

Multiple Output Dimensions

Here we will just briefly extend the above to the case where we have multiple output dimensions (multiple targets e.g. predicting both the price and the size of a house). Now this becomes a full matrix problem, where we have a weight matrix which I will use \(\mathbf{\Theta}\) for, and a target matrix \(\mathbf{Y}\). These are defined as:

\[ \mathbf{\Theta} = \begin{bmatrix} w_{11} & w_{12} & \cdots & w_{1k} \\ w_{21} & w_{22} & \cdots & w_{2k} \\ \vdots & \vdots & \ddots & \vdots \\ w_{(n+1)1} & w_{(n+1)2} & \cdots & w_{(n+1)k} \end{bmatrix}, \quad \mathbf{Y} = \begin{bmatrix} y_{11} & y_{12} & \cdots & y_{1k} \\ y_{21} & y_{22} & \cdots & y_{2k} \\ \vdots & \vdots & \ddots & \vdots \\ y_{N1} & y_{N2} & \cdots & y_{Nk}\end{bmatrix} \]

Where \(k\) is the number of output dimensions and \(N\) is the number of data points. I won't go through the full derivation, but the solution is given by:

\[ \mathbf{\Theta} = (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{Y} \]

Non-Linear Basis Functions

In all of the above examples we have only looked at the Linear Regression model, which, as the name suggests, can only model linear relationships between the input and output variables. However, we can extend this model to handle non-linear relationships by using basis functions. The idea is to transform the input features into a higher dimensional space using a basis function and then apply linear regression in this new space.

The most basic example of this is the polynomial basis function, where we take out input function and raise it to some power. For example, if we have a single input feature \(x\) and we want to use a polynomial basis function of degree 2, we would transform our input as follows:

\[ \phi(x) = [1, x, x^2] \]

This allows us to model quadratic relationships between the input and output variables, which we couldn't do with linear regression alone.

We can also do this with more input features for example if we have \(M\) input features and \(k\) output features, we can define a second order polynomial basis function as follows:

\[ \phi(\mathbf{x}) = [1, x_1, x_2, \cdots, x_M, x_1^2, x_2^2, \cdots, x_M^2, x_1x_2, x_1x_3, \cdots, x_{M-1}x_M] \]

\(1\) is the bias term, x_n are the original linear features, \(x_n^2\) are the squared features and \(x_n x_m\) are the interaction features. We could equally define this without the interaction features, but that would limit the surface (the resultant function space) to seperable surfaces along each feature dimension and would not fully define a full quadratic surface.

Like before, we can define the design matrix and target matrix in this new feature space and apply the same least squares solution to find the optimal parameters. The design matrix is given by:

\[ \mathbf{\Phi} = \begin{bmatrix} 1 & x_{11} & x_{12} & \cdots & x_{1M} & x_{11}^2 & x_{12}^2 & \cdots & x_{1M}^2 & x_{11}x_{12} & \cdots & x_{1(M-1)}x_{1M} \\ 1 & x_{21} & x_{22} & \cdots & x_{2M} & x_{21}^2 & x_{22}^2 & \cdots & x_{2M}^2 & x_{21}x_{22} & \cdots & x_{2(M-1)}x_{2M} \\ \vdots & \vdots & \vdots & \ddots & \vdots & \vdots & \vdots & \ddots & \vdots & \vdots & \ddots & \vdots\\ 1 & x_{N1} & x_{N2} & \cdots & x_{NM} & x_{N1}^2 & x_{N2}^2 & \cdots & x_{NM}^2& x_{N1}x_{N2} & \cdots & x_{N(M-1)}x_{NM} \end{bmatrix} \]

This is extremely long so we can define it more concisely as:

\[ \mathbf{\Phi} = \begin{bmatrix} \boldsymbol{\phi}(\mathbf{x}_1) \\ \boldsymbol{\phi}(\mathbf{x}_2) \\ \vdots \\ \boldsymbol{\phi}(\mathbf{x}_N) \end{bmatrix} \]

The target matrix is defined as before, but the weight matrix is now in the transformed feature space so has more dimensions. We need a weight for each of the new features so lets calculate the dimensions for each data point.

We have \(1\) bias term, then we have \(M\) linear features, and also \(M\) squared features. We also have the interaction terms and there are \(\frac{M(M-1)}{2}\) of those. So the total weights we need per data point is:

\[D = 1 + 2M + \frac{M(M-1)}{2}\].

And as such the weight matrix is \(\mathbf{\Theta} \in \mathbb{R}^{D \times N}\), which I won't write out again but it is essentially the same as in the multiple output section just with some more dimensions.

The solution to the least squares problem is essentially the same before, just with our new design matrix:

\[ \mathbf{\Theta} = (\mathbf{\Phi}^\top \mathbf{\Phi})^{-1} \mathbf{\Phi}^\top \mathbf{Y} \]

There are also many other basis functions we could use, for example:

  • Radial basis functions (RBFs)

  • Fourier basis functions

  • Wavelet basis functions

I won't go into the details of these, but the general idea is the same. We transform our input features into a new feature space with the basis function and then perform linear regression in this space.

Regularisation

Sometimes we might find that the model we chose is actually too powerful/complex for the task at hand. And as a result we end up with a model that fits the training data very well but performs poorly on unseen data. We call this overfitting, and it's a topic that will come up often in this course as it's a constant challenge for researchers. We need to try to fit the data best but still make sure we aren't just fitting noise and we can generalise to data we haven't seen yet. A good example of this would be in finance:

Lets say we have a dataset of prices for a particular instrument (e.g S&P 500) and we want to predict the future price of this instrument based on its past prices. We could use a linear regression model with a basis function to model the relationship between the past prices and the future price. We might even find that we can fit it very well. However, when we go to test our model in the next year we find that it performs very poorly. In this case, we know that the model is overfit and is not generalising to the new price movements.

So how do we combat this?

We still want to use our powerful model, but just constrain it in some way so that it doesn't overfit our historical data. This is where regularisation comes in.

What we do is add a penalty term to our least squares function that penalises large weights. This encourages the model to find a solution with smaller weights, which is less likely to overfit the data. The two most common types of regularisation are L1 regularisation (Lasso) and L2 regularisation (Ridge).

L1 Regularisation (Lasso)

L1 regularisation uses the L1 norm of the weight vector as the penalty term. In other words, we add the absolute values of the weights to our loss function:

\[ \mathbf{L}(\mathbf{\theta}) = \|\mathbf{y} - \mathbf{X} \mathbf{\theta}\|^2 + \lambda \|\mathbf{\theta}\|_1 \]

and there is no closed form solution for this one, so we must use an iterative algorithm like gradient descent to find the approximate solution.

L2 Regularisation (Ridge)

L2 regularisation uses the L2 norm of the weight vector as the penalty term. In other words, we add the squared values of the weights to our loss function:

\[ \mathbf{L}(\mathbf{\theta}) = \|\mathbf{y} - \mathbf{X} \mathbf{\theta}\|^2 + \lambda \|\mathbf{\theta}\|_2^2 \]

and there is a closed form solution for this one:

\[ \mathbf{\theta} = (\mathbf{X}^\top \mathbf{X} + \lambda \mathbf{I})^{-1} \mathbf{X}^\top \mathbf{y} \]

where $\mathbf{I}$ is the identity matrix.

This also has a nice property that means that the matrix we are inverting is always invertible, which is not the case for the unregularised least squares solution.

The Bias-Variance Tradeoff

In the same spirit as regularisation, we often have to make a tradeoff between how well our model fits the training data and how well it generalises to unseen data. This is the bias-variance tradeoff.

Let's say the data is generated by some function \(f(x)\) and we have some noise \(\epsilon\) that is added to the output. So our data points are generated by the following process:

\[ y = f(x) + \epsilon, \quad \mathbb{E}[\epsilon] = 0, \quad \text{Var}(\epsilon) = \sigma^2 \]

We train a model \(\hat{f}(x)\) using dataset \(D\), which means that \(\hat{f}(x)\) is a random variable that depends on the dataset we used to train it. So therefore, if we care about the expected error of our model on some new data generated by the same process, we need to take the expectation over the dataset as well as the noise:

\[ \mathbb{E}_{D, \epsilon}[(y - \hat{f}(x))^2] = \mathbb{E}_{D}[(f(x) - \hat{f}(x))^2] + \sigma^2 \]

Essentially what we are doing here is taking the average error of our model across:

  1. All different possible datasets \(D\)

  2. All different possible noise values \(\epsilon\)

This instantly tells us something about the expected error. One part is unavoidable. The noise in the generation process. The other part is the error due to our model not being able to capture the true function \(f(x)\) perfectly.

We can futher decompose the error due to our model into two parts: bias and variance. To do that we first need to define a new quantity. The mean prediction across datasets:

\[ \bar{f}(x) = \mathbb{E}_D[\hat{f}(x)] \]

Then we can have the following decomposition. We can say that the deviation of our model from the average model is \(\hat{f}(x) - \bar{f}(x)\). And then the deviation of the average model from the true function is \(\bar{f}(x) - f(x)\). So we can write:

\[ \mathbb{E}_{D}[(f(x) - \hat{f}(x))^2] = \mathbb{E}_{D}[((f(x) - \bar{f}(x)) + (\bar{f}(x) - \hat{f}(x)))^2] \]

It seems counter intuitive at first that we are adding and subtracting the same quantity, but once you expand the expression you are left with:

\[ \mathbb{E}_{D}[(f(x) - \hat{f}(x))^2] = (f(x) - \bar{f}(x))^2 + \mathbb{E}_{D}[(\hat{f}(x) - \bar{f}(x))^2] \]

Where the first term is the bias squared and the second term is the variance. So overall we have the following decomposition:

\[ \mathbb{E}_{D}[(f(x) - \hat{f}(x))^2] = \text{bias}^2 + \text{variance} + \sigma^2 \]

So what does this mean?

When you train a model you typically only see one dataset. But there are many datasets you could have seen. So imagine training your model many times on those different datasets.

At a high level, if you take the average of all of those models you get the average model \(\bar{f}(x)\). The bias is the error of this average model from the true function. So it essentially tells you if your model is systematically wrong in some way. For example, you are trying to fit a linear model to data that is generated by a quadratic function. In this case, the bias will be large because the average model is a straight line and the true function is a curve.

On the other hand, the variance is a measure of the instability of your model across the different datasets. Essentially it is the error of your model from the average. So if your model is very sensitive to the specific dataset you trained it on, then the variance will be large. In the example of the linear model, the variance will also be large as it will be sensitive to the specific data points it sees.

The aim of the machine learning researcher is to find a model that has both a low bias and a low variance, which is often a tradeoff.

An example

Lets look at some data that we know the generation process for (in practice you may have an idea or none at all but rarely will you know exactly). And lets take our linear regression model - with and without basis functions - and try to fit it to the data.

Below we have some data generated by a cubic function with some noise added to it. We then fit 3 regression models to this data:

  1. Linear regression (no basis function)

  2. Cubic regression (with \(x^3\) basis functions)

  3. Ninethic regression (with \(x^9\) basis functions)

First the data on its own, you can see it is quite noisy but there is a clear cubic curve.

Now we fit a linear regression model to this data.

In this case is a high bias model as it is not powerful enough to capture the true relationship, but it is low variance as it is not sensitive to the specific data points. Which we can see by generating another dataset from the same process and fitting the same model to it. We get a similar line of best fit.

Next we fit a cubic regression model, the correct model for the data.

As you can see this is a much better fit to the data, it has a much lower bias as it is able to capture the true relationship. However, it has a much higher variance as it is more sensitive to the specific data points. You can see that the difference in the fitted curve between the two datasets is much larger than for the linear model.

Finally we fit a ninethic regression model, which is much more powerful than the true model.

As you can see this fits the data perfectly, it has a very low bias. However if we go and look at the fit for the second dataset we see that it is a very different curve, which means it has a very high variance.

In general, the aim is balance the bias and variance to find a model that fits the data well but also generalises to unseen data.
